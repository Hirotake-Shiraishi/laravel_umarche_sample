## 2. 修正内容（評価資料で「正確に実装」とされた箇所）

### 2.1 CartAddRequest（CartController::add() のバリデーション）

**ファイル:** `app/Http/Requests/CartAddRequest.php`

- **目的:** カート追加時に `quantity`・`product_id` をそのまま使うと、負の数や存在しない ID で孤立レコードが作れる問題を防ぐ。
- **実装:**
  - `quantity` => `required|integer|min:1|max:99`
  - `product_id` => `required|integer|exists:products,id`
- **利用箇所:** `app/Http/Controllers/User/CartController.php` の `add(CartAddRequest $request)` で注入。

**コメント追記:** docblock に「指摘#5 / 課題4: バリデーション強化」を明記。CartController::add() の docblock にも同様に【課題4】を追記。

---

### 2.2 OwnersController::update() のバリデーション

**ファイル:** `app/Http/Controllers/Admin/OwnersController.php`

- **目的:** update() にバリデーションがなく、name/email が空でも保存でき、メール形式・一意性チェックもない。また password が空のときに `Hash::make('')` で空ハッシュが保存されていた問題を解消。
- **実装:**
  - `$request->validate([...])` を追加。
    - `name` => `required|string|max:255`
    - `email` => `required|email|max:255|unique:owners,email,{$id}`（自分自身の ID を除外）
    - `password` => `nullable|string|confirmed|min:8`
  - パスワードは「入力があるときだけ」更新: `if ($request->filled('password')) { $owner->password = Hash::make($request->password); }`。未入力の場合は `$owner->password` を触らない。

**コメント追記:** update() の docblock に「指摘#4 / 課題4: バリデーション強化 修正済み」および【課題4】を明記。

---

### 2.3 Product スコープの return 修正（課題4 評価対象）

**ファイル:** `app/Models/Product.php`

- **目的:** else で `return;` だと null を返しメソッドチェーンが切れる。想定外の `$sortOrder` でもチェーンが切れないようにする。
- **実装:**
  - **scopeSelectCategory:** `else { return; }` → `else { return $query; }`
  - **scopeSearchKeyword:** `else { return; }` → `else { return $query; }`
  - **scopeSortOrder:** 関数末尾に `return $query;` を追加（想定外値でもチェーンが切れないようにする）。

**コメント追記:** 各スコープの docblock に「指摘#7 / 課題4 評価対象 修正済み」および【課題4】を明記。
