## 2. 修正内容

### 2.1 CartController::checkout()（悲観的ロック）

**ファイル:** `app/Http/Controllers/User/CartController.php`

- **在庫チェックの位置**
  - **修正前:** トランザクション外で `Stock::where(...)->sum('quantity')` を実行し、不足ならリダイレクト。その後 `DB::transaction` 内で在庫減算と Stripe 呼び出し。
  - **修正後:** 在庫チェックをトランザクション内に移動。`$lineItems` は在庫数に依存しないため、トランザクション外で組み立てたまま。

- **lockForUpdate（悲観的ロック）**
  - トランザクション内で、全商品について `Stock::where('product_id', $product->id)->lockForUpdate()->sum('quantity')` で在庫取得。
  - `$product->pivot->quantity > $quantity` のとき `throw new \Exception('在庫不足')` でロールバック。
  - 全件 OK のときのみ、同じトランザクション内で在庫減算（`Stock::create` で type reduce）→ Stripe Checkout Session 作成。

- **例外処理**
  - `catch (\Throwable $e)` で `$e->getMessage() === '在庫不足'` のときは `redirect()->route('user.cart.index')->with(['message' => '在庫不足です。', 'status' => 'alert'])`。
  - それ以外（Stripe 失敗等）は従来どおり「決済の開始に失敗しました。」でリダイレクト。

- **コメント**
  - 上記の「何を・どのように」変更したかを、該当箇所にコメントで記載済み。

### 2.2 Feature テスト（StockCheckoutTest）

**ファイル:** `tests/Feature/StockCheckoutTest.php`（新規）

- **テストケース**
  1. **test_product_with_one_stock_appears_in_item_list**  
     在庫1個の商品を用意し、認証ユーザーで商品一覧（`/`）にアクセスしたときにその商品が表示されることを検証。  
     （scopeAvailableItems の `having('quantity', '>', 0)` により在庫1個でも一覧に出ることを確認。）

  2. **test_checkout_redirects_to_cart_index_when_insufficient_stock**  
     在庫1個の商品をカートに2個入れた状態でチェックアウトにアクセスし、`user.cart.index` へリダイレクトされ、セッションに「在庫不足です。」が入ることを検証。  
     （checkout() 内の lockForUpdate → 在庫チェック → 不足時 Exception → catch でリダイレクトの流れを確認。）

- **テストデータ（seedItemListDependencies）**
  - `Product::availableItems()` が JOIN する shops / secondary_categories / images を用意するため、テスト内で `DB::table()->insert()` により次の順で挿入。
  - 順序: owners → primary_categories → secondary_categories → shops → images  
  - RefreshDatabase 利用時に Seeder のみだと外部キー制約で失敗したため、テスト内で直接挿入する形に統一。
